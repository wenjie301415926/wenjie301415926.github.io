为什么要搭建个人博客

  • 自己的才是最好的：完全自主的控制权；个性定制；逼格高
  • 可以学到不少新知识：基础的网页开发；git分布式管理系统；站长知识

具体教程

我写的这篇教程，不会有手把手一步一步的详细讲解，因为各个步骤隔一段时间了就会有大小不一的变化（我在学习的过程中就碰到不少因为教程时间过去太久而实际已经改变的情况），所以这里只说大致步骤，具体细节自己多查资料多实践。

建独立的个人博客最关键就是”域名+主机”

域名就是大家经常见到的主站网址，是需要购买的（别急，不贵）；主机则是一个24小时开机的服务器，这样才能随时访问你的网站，这里我们把网站文件都托管到github，使用方便而且免费。

解决完这两个问题后，再要考虑的就是：

  1. 如何把域名和github绑定起来
  2. 怎么搭建网站
  3. 如何把本地写好的网页资料上传到github云端
  4. 如何写文章
  5. 如何优化网站

域名部分

域名购买可以选狗爹GoDaddy，支持支付宝。

不同后缀类型的价格会有一定差别，但并不是太大，一般在RMB50~100/年，正式支付之前去搜一搜GoDadyy优惠码，能够减少一二十元。
另外注册狗爹账户时，密码一定要记清，不然找回密码时挺麻烦的

GoDaddy购买域名详细教程

由于某个我国特殊国情的原因，我们必须多做些事情，把域名解析服务迁移到国内比较稳定的服务商处。我们选择DNSPod的服务。注册登录之后，只需三步即可搞定：

  1. 首先添加域名记录，可参考DNSPod的帮助文档：https://www.dnspod.cn/Support
  2. 在DNSPod自己的域名下添加一条A记录，地址就是Github Pages的服务IP地址：207.97.227.245
  3. 在域名注册商处修改DNS服务:去Godaddy修改Nameservers为这两个地址：f1g1ns1.dnspod.net、f1g1ns2.dnspod.net。如果你不明白在哪里修改，可以参考这里

Git与GitHub部分

1.首先介绍一下Git和GitHub

git百度百科
GitHub维基百科

github和git本身是程序员开发软件（特别是团队合作开发）时的一大利器，但这与我们无关。我们只需要知道它方便建站，我们把博客的所有内容都托管在GitHub上，它相当于就是我们的主机。Git则是相当于一个电脑客户端，把我们在本地写的文章、构建的数据上传到GitHub云端。github与本地git的关系，粗浅地可理解为百度云盘与百度云管家，一个是云端，一个是本地客户端，都能管理组织你的文件资料。

2.下载安装Git客户端

windows的git客户端（最近好像被墙了，再给一个云盘分享)

git学习（非常重要）
git内容太多，我没这个能力讲授。分享一篇git教程，是我见过的最好的一篇，半天内基本了解，比官网介绍的更容易上手学习——Git教程。从开头至少要看到分支管理，才足够你建站所需基本知识。

3.注册GitHub帐号

GitHub官方网站

注册好账户后，新建一个与账户名相同的repository。若你的账户名为yourusername,则该repository命名为yourusername.github.io。在新建的库里新建一个CNAME的文件，里面写入你得域名。

其实此时你在这个repository里写入index文件就能访问了。例如你可以用记事本写下一个html文件,内容随便复制一篇网页的网页源代码，访问你的域名即可看到该网页。

不过在仓库里直接写显然不方便，所以要把本地的git和云端的github联系起来，以后在本地写好后再上传上去。

4.让本地git项目与远程的github建立联系——配置SSH keys

如何让本地git项目与远程的github建立联系呢？配置SSH keys。

首先我们需要检查你电脑上现有的ssh key。打开git命令窗，输入以下命令

cd ~/.ssh

然后系统会要你输入密码。在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。

注意：输入密码的时候没有*字样的，你直接输入就可以了。最后看到这样的界面，就成功设置ssh key了：

在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。

  1. 打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
  2. 登陆github系统。点击右上角的 Account Settings—->SSH Public keys —-> add another public keys

  3. 把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了

    可以输入下面的命令，看看设置是否成功，git@github.com的部分不要修改：

    ssh -T git@github.com
